# FlAI Brand Reference

> This file is the single source of truth for all FlAI agents creating content, graphics, or copy.

## Identity

**Product:** FlAI (pronounced "fly")
**Website:** getflai.dev
**GitHub:** github.com/FlutterAiChat/flutter-ai-chat-components
**Tagline:** shadcn/ui for Flutter AI chat
**One-liner:** Drop-in AI chat components for Flutter. You own the source code.

## Positioning

FlAI is a component library, not a package. Developers install components as source code via a CLI (`flai add`), just like shadcn/ui does for React. This means:
- No version lock-in — you own every line
- Full customization — not fighting an API surface
- Copy-paste philosophy — the code lives in YOUR project

**Key differentiator vs competitors:** They own your code. We give you your code.

## Voice & Tone

- Developer-first. Write like you're explaining to a smart friend, not lecturing.
- Confident but not arrogant. We know this is good. We don't need to trash others.
- Concise. Every word earns its place. No filler, no corporate speak.
- Slightly edgy. Opinionated takes are fine. "Flutter AI chat packages are broken" is on-brand.
- Show, don't tell. Code snippets > feature lists. GIFs > screenshots. Demos > promises.

**Alex's voice:** Casual, direct, uses "lol" and "ngl", drops articles sometimes, writes in fragments. Think engineer-tweets, not press releases. Reference `/sessions/vigilant-ecstatic-johnson/mnt/.auto-memory/user_voice_style.md` for the full voice profile.

## Typography

- Headlines: Clean sans-serif (Inter, Geist, or similar)
- Code: JetBrains Mono or Fira Code
- Body: Same as headlines, lighter weight
- In Canva: Use bold sans-serif for headlines, monospace for code snippets

## Visual Style

- Dark mode first — all hero images and social cards use dark backgrounds
- Terminal/code aesthetic — showing real Flutter code is always on-brand
- Minimal, high contrast — no gradients, no busy patterns
- Linear/Vercel energy — clean, developer-tool aesthetic
- Chat bubbles are a signature visual element — use them in graphics when possible

## Content Themes (Campaign)

- Week 1: "Flutter AI chat is broken" — pain, frustration, relatable problems
- Week 2: "We built FlAI" — the reveal, demos, first looks
- Week 3-4: "Building in public" — tutorials, live coding, early access
- Week 5: "v0.1.0 is live" — launch, celebration, community

## What NOT To Do

- Don't use light/bright color schemes in marketing (save that for the docs site)
- Don't call FlAI a "package" — it's a "component library" or "toolkit"
- Don't use corporate language ("leverage", "synergize", "ecosystem")
- Don't compare to specific competitors by name in public posts (unless it's an objective comparison)
- Don't use stock photos — ever
- Don't use AI-generated images of people
