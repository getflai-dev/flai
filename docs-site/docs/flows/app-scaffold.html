App Scaffold

Production app shell wiring auth, onboarding, chat, and sidebar flows with GoRouter. One command installs everything.

Installation

$ flai add app_scaffold

What Gets Installed

Running flai add app_scaffold auto-installs 5 bricks and the go_router dependency:

auth_flow onboarding_flow chat_experience sidebar_nav app_scaffold

The full dependency tree:

// app_scaffold
//   ├── auth_flow
//   ├── onboarding_flow
//   ├── chat_experience
//   ├── sidebar_nav
//   └── go_router (added to pubspec.yaml)

Architecture

FlaiApp is a StatefulWidget that sets up the entire app shell with GoRouter navigation:

// FlaiApp
//   └── FlaiProviders (InheritedWidget)
//       └── FlaiTheme
//           └── MaterialApp.router
//               └── GoRouter
//                   ├── /splash     → SplashScreen
//                   ├── /auth       → AuthFlowScreen
//                   ├── /onboarding → OnboardingFlowScreen
//                   └── /home       → HomeScreen (chat + sidebar)

Auth state redirects are handled by AuthStateNotifier, which listens to AuthProvider.authStateChanges and triggers GoRouter redirects:

  • Unauthenticated users are redirected to /auth
  • New users (first login) are redirected to /onboarding
  • Authenticated users land on /home

Configuration

AppScaffoldConfig composes all flow configurations into a single entry point:

PropertyTypeDefaultDescription
authProvider AuthProvider required Authentication provider implementation.
aiProvider AiProvider? null AI chat streaming provider.
storageProvider StorageProvider? InMemoryStorageProvider Conversation persistence provider.
voiceProvider VoiceProvider? null Voice transcription and synthesis provider.
theme FlaiThemeData FlaiThemeData.dark() Theme data applied to the entire app.
authFlowConfig AuthFlowConfig? null Configuration for the auth flow screens.
onboardingConfig OnboardingConfig? null Configuration for the onboarding pipeline.
chatExperienceConfig ChatExperienceConfig? null Configuration for the chat experience widgets.
sidebarConfig SidebarConfig? null Configuration for the sidebar navigation.

Providers

FlaiProviders is an InheritedWidget that exposes all 4 provider interfaces from anywhere in the widget tree:

// Access providers from any widget
final providers = FlaiProviders.of(context);

providers.authProvider;     // AuthProvider
providers.aiProvider;       // AiProvider?
providers.storageProvider;  // StorageProvider
providers.voiceProvider;    // VoiceProvider?

Usage

void main() {
  runApp(FlaiApp(
    config: AppScaffoldConfig(
      authProvider: MyAuthProvider(),
      aiProvider: MyAiProvider(),
      theme: FlaiThemeData.dark(),
      authFlowConfig: AuthFlowConfig(
        showApple: true,
        showGoogle: true,
      ),
      onboardingConfig: OnboardingConfig(steps: [
        OnboardingStep.naming(suggestions: ['Nova', 'Atlas']),
        OnboardingStep.reveal(),
      ]),
    ),
  ));
}
